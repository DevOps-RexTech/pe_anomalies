🔵 Why EKS — Honest Reasons

  • RAG Orchestrator is a long-running stateful service — Lambda 15min limit doesn't fit
  • Prometheus + Grafana are Kubernetes-native — this is genuinely their home
  • EKS gives you HPA — auto-scale RAG pods during EOD crunch automatically
  • Fluent Bit as a DaemonSet collects logs from every node automatically
  • GitOps via ArgoCD — git push = deployment, audit trail for compliance

🟡 Why Lambda — Also Honest

  • Feed validation is event-driven — fires on S3 arrival, idle otherwise
  • Anomaly detection runs per-file, not continuously — Lambda is cheaper
  • Alert dispatcher is a simple HTTP call — no reason to containerise it
  • SNS fan-out is fully managed — no infrastructure to maintain
  • Cost: Lambda for these functions = near zero vs running extra pods on EKS
